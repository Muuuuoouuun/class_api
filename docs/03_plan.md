# ClassIn 서드파티 자동화 컨설팅 프로젝트
## 실행 계획

---

## 1. 코어 엔진 (모든 MVP의 기반)

### 수업·숙제 자동 생성기

**한 줄 정의**
> 스케줄표 + 진도표를 입력하면 Claude가 파싱하여 ClassIn에 수업과 숙제를 일괄 세팅한다.

**흐름**
```
[스케줄 CSV or 폼 입력]
        ↓ Claude 파싱 (날짜·시간·과목·진도 구조화)
[ClassIn CED API]
        ↓ 수업 생성 + 숙제 등록
[Notion DB]
        ↓ ClassIn 반환 ID값 저장
```

**이 엔진 없이는 나머지 MVP 불가. 가장 먼저 구현.**

## 2. MVP 3개

---

### MVP 1. 데이터 수집 + 숙제 미제출 푸시 알림

**목적**
숙제 안 한 학생을 일일이 확인하고 연락하는 시간 제거.
수업 종료 후 30분 내 자동 알림.

**흐름**
```
ClassIn Webhook (After-Class, 수업 종료 후 20분 내)
        ↓
Notion DB 자동 적재
(학생별 숙제 제출 여부, 출석, 참여 데이터)
        ↓
Claude: 미제출자 추출 + 알림 문구 생성
(학생·학부모별 맞춤 문구)
        ↓
카톡 알림 발송 (알리고/솔라피)
```

**구현 요소**
| 요소 | 구현 방식 | 비고 |
|------|-----------|------|
| Webhook 수신기 | FastAPI + Cloudflare Tunnel | 로컬 학원 PC |
| DB 적재 | Notion API | 수업 기록 DB |
| 미제출 추출 | Claude 프롬프트 | After-Class 숙제 데이터 기반 |
| 카톡 발송 | 알리고 or 솔라피 API | MVP 단계: 예시 문구만 |

**완성 기준**
수업 종료 후 30분 내에 미제출자 카톡이 자동 발송되면 완성.

**구현 난이도**: ★★☆

---

### MVP 2. 주간 학생 리포트 자동 생성

**목적**
매주 학부모 상담 자료 만드는 2~3시간 제거.
ClassIn에서 제공하지 않는 학생 참여도 분석 가치 창출.

**흐름**
```
Notion DB (누적 데이터)
(출석 + 참여도: 손들기/트로피/카메라/Poll + 숙제 제출률)
        ↓
매주 금요일 Claude 실행
        ↓
학생별 개인화 분석 (학생마다 다른 리포트)
        ↓
Notion 리포트 페이지 자동 생성
+ 학부모 카톡 발송용 문구 생성
```

**리포트 포함 항목**
- 이번 주 출석률 / 지각 횟수
- 수업 참여도 점수 (손들기·트로피·카메라 통합)
- 숙제 제출 현황
- Claude 종합 코멘트 (학생 상태 맞춤 문구)
- 지난 주 대비 변화 추이

**완성 기준**
5명 샘플 학생 데이터로 각각 다른 리포트가 자동 생성되면 완성.

**구현 난이도**: ★☆☆ (데이터 적재 완료 시)

## 3. 구현 타임라인

```
Week 1          Week 2          Week 3          Week 4
────────────────────────────────────────────────────────
[코어 엔진]     [MVP 1]         [MVP 2]         [데모·영업]
수업·숙제       Webhook 수신    리포트 생성     데모 영상 +
자동 생성       + 미제출 알림   자동화          제안서 완성
```

### Week 1 상세: 코어 엔진

| 일차 | 작업 |
|------|------|
| Day 1~2 | ClassIn 샘플 Webhook 페이로드 확보 + Notion DB 스키마 설계 |
| Day 3~4 | FastAPI Webhook 수신기 + Cloudflare Tunnel 연결 |
| Day 5~7 | CED API 래퍼 작성 (수업·숙제 생성) + Claude 파싱 프롬프트 |

### Week 2 상세: MVP 1

| 일차 | 작업 |
|------|------|
| Day 8~9 | After-Class 샘플 페이로드 → Notion 적재 검증 |
| Day 10~11 | Claude 미제출자 추출 프롬프트 + 카톡 문구 생성 |
| Day 12~14 | 알리고/솔라피 연동 테스트 + 자동 트리거 설정 |

### Week 3 상세: MVP 2

| 일차 | 작업 |
|------|------|
| Day 15~16 | 누적 데이터 기반 Claude 분석 프롬프트 설계 |
| Day 17~18 | 5명 샘플 페르소나 데이터 세팅 + 리포트 차별화 검증 |
| Day 19~21 | Notion 리포트 페이지 자동 생성 + 학부모 문구 템플릿 |

### Week 4 상세: 데모·영업 무기화

| 일차 | 작업 |
|------|------|
| Day 22~23 | 가상 학원 데모 시나리오 완성 (5개 페르소나) |
| Day 24~25 | Loom으로 3~5분 데모 영상 촬영 |
| Day 26~28 | 제안서 PDF 작성 (가치·가격·도입 절차) |

## 7. 레퍼런스 데이터 구조 (Notion DB 스키마 초안)

### 학생 Master DB
| 필드 | 타입 | 설명 |
|------|------|------|
| 학생명 | Text | - |
| ClassIn ID | Text | CED API 반환값 |
| 반 | Relation | 반 DB 연결 |
| 학부모 연락처 | Phone | 카톡 알림 수신 번호 |
| 등록일 | Date | - |
| 상태 | Select | 재원/휴원/퇴원 |

### 수업 기록 DB
| 필드 | 타입 | 설명 |
|------|------|------|
| 학생 | Relation | 학생 Master 연결 |
| 수업일시 | Date | - |
| 출석 여부 | Select | 출석/지각/결석 |
| 손들기 횟수 | Number | After-Class 데이터 |
| 트로피 수 | Number | After-Class 데이터 |
| 카메라 시간(분) | Number | After-Class 데이터 |
| 숙제 제출 | Checkbox | - |
| ClassIn 수업 ID | Text | CED API 반환값 |

### 리포트 DB
| 필드 | 타입 | 설명 |
|------|------|------|
| 학생 | Relation | 학생 Master 연결 |
| 리포트 기간 | Date | 주간/월간 |
| 리포트 내용 | Text | Claude 생성 분석 |
| 학부모 발송 문구 | Text | 카톡 발송용 |
| 발송 여부 | Checkbox | - |
| 발송일시 | Date | - |
