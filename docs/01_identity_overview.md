# ClassIn 서드파티 자동화 컨설팅 프로젝트
## 정체성 및 전체 흐름

---

## 1. 프로젝트 정의

### 한 줄 정의
> ClassIn API 기반 자동화 엔진을 학원에 로컬 설치하고, Claude 기반 지능형 자동화를 얹어 학원 운영 전반의 가치를 높이는 컨설팅 서비스.

### 포지셔닝
- **형태**: SaaS가 아닌 **로컬 설치 + 컨설팅 모델**
- **공급자**: MOON (ClassIn 한국 지사 담당자 겸 컨설턴트)
- **수요자**: ClassIn을 사용하는 학원 원장 및 교사
- **차별점**: ClassIn 기본 기능으로 해결되지 않는 오프라인 영역까지 통합, Claude를 범용 자동화 엔진으로 활용

### 왜 로컬 설치인가
| 항목 | SaaS | 로컬 설치 + 컨설팅 |
|------|------|-------------------|
| API 키 보관 | MOON 서버에 저장 | 학원이 직접 보관 |
| 개인정보 책임 | MOON에게 있음 | 학원에게 있음 |
| 서버 운영 | MOON 24/7 | 학원 책임 |
| 법적 복잡성 | 높음 | 낮음 |
| 인프라 비용 | 월 고정 | 사실상 0 |

## 2. ClassIn API 구조 이해

### 패키지 구분
- **Basic API**: 관리자만 API 사용. CED + Data 구독 = **무료**
- **Advanced API**: 교사·학생까지 SSO 등 API 사용. 유료.
- **Bespoke**: 가상 계정 등 맞춤형. 별도 협의.

### 연동 형태 3가지
1. **CED (Create/Edit/Delete)**: 계정·코스·레슨·드라이브 CRUD
2. **Data Subscription**: Webhook으로 44종 데이터 수신 (1기관 1 엔드포인트)
3. **Bespoke**: 가상 계정 부가 포함 맞춤형

### 핵심 기술 제약
- ClassIn은 **PUSH(Webhook) 방식만** 제공. GET 폴링 불가.
- After-Class 데이터는 수업 종료 후 **20분 이내** 전송.
- LMS 성적 조회 API 없음.
- 대시보드 동시 조작 금지 (데이터 충돌 위험).

### Webhook을 로컬에서 받는 방법
- **개발 단계**: ngrok 또는 Cloudflare Tunnel로 로컬 노출
- **운영 단계 (권장)**: Supabase Edge Function (무료) → DB 적재 → 로컬 앱이 pull

## 3. 전체 아키텍처

```
┌───────────────────────────────────────────────────────────┐
│                   학원 운영 자동화 플랫폼                   │
├───────────────────────────────────────────────────────────┤
│                                                           │
│  [Layer 4: 출력/커뮤니케이션]                             │
│   카톡 알림 · 학부모 리포트 · 대시보드 · 문서 생성        │
│         ▲                                                 │
│  [Layer 3: 지능화]                                        │
│   Claude 자동화 (분석 · 리포트 · 커뮤니케이션 문구 생성)  │
│         ▲                                                 │
│  [Layer 2: 데이터 통합 허브]                              │
│   Notion DB (단일 진실원)                                 │
│         ▲                                                 │
│  [Layer 1: 데이터 수집]                                   │
│   ClassIn Webhook | 오프라인 입력 (QR/CSV) | 외부 시스템  │
│                                                           │
└───────────────────────────────────────────────────────────┘
         ↓ 로컬 설치 + 컨설팅 모델로 학원에 공급
```

## 4. 기능 모듈 전체 지도

### 코어 모듈 (ClassIn API 기반)
| 모듈 | 기반 | 가치 |
|------|------|------|
| 계정·반·수업 일괄 관리 | CED | 학기 초 반복 작업 제거 |
| 온라인 출석·지각 자동 수집 | Data 구독 | 수기 입력 제거 |
| 학생 참여도 리포트 | Data 구독 (손들기/트로피/카메라/Poll) | ClassIn에 없는 가치 |
| 교사 수업 품질 리포트 | Data 구독 | 교사 평가 데이터화 |

### 부가 모듈 (학원 운영 전반)
| 모듈 | 핵심 기능 | 우선순위 |
|------|-----------|----------|
| **출결 통합** | QR 오프라인 + 온라인 merge, 지각 카톡 | ★★★ |
| **시험·성적 통합 DB** | 모의고사·내신·오프라인 시험, Claude OCR | ★★★ |
| **학부모 커뮤니케이션** | 주간 리포트, 카톡 알림톡, 상담 자료 자동화 | ★★★ |
| **통합 대시보드** | 원장/교사 뷰, 이상 징후 자동 감지 | ★★ |
| **업무 자동화** | 증명서 자동 생성, 일정 관리, 퇴원 예측 | ★★ |
| **마케팅 확장** | 성과 증빙, SNS 콘텐츠, 후기 요청 자동화 | ★ |

## 5. 비즈니스 모델

### 수익 구조
- **초기 설치비**: 세팅 + 교육 (1회성)
- **월 유지보수비**: 업데이트 + 문의 대응 (선택)
- **추가 모듈**: 기능 커스터마이징 요청 시 별도 견적

### 패키지 티어
| 티어 | 포함 모듈 | 대상 |
|------|-----------|------|
| **Starter** | ClassIn 자동화(계정·출석) + 주간 리포트 | 소형 학원 |
| **Standard** | + 오프라인 출결 + 카톡 알림 + 대시보드 | 일반 학원 |
| **Pro** | + 모의고사/내신 성적 통합 + 상담 자료 자동화 | 입시 전문 학원 |
| **Custom** | 학원별 맞춤 | 프랜차이즈·특수 요구 |

## 6. 기술 스택

```
[수집]     Cloudflare Tunnel + FastAPI (Webhook 수신)
           Supabase Edge Function (운영 단계)
           ↓
[저장]     Notion Database (학원별 통합 DB)
           SQLite (로컬 백업)
           ↓
[지능화]   Claude API (분석·리포트·문구 생성·OCR)
           ↓
[출력]     Notion 페이지 (리포트 저장)
           카카오 알림톡 (알리고/솔라피 릴레이)
           Streamlit (로컬 대시보드, 선택)
```

## 7. 성공 조건 & 리스크

### 성공 조건
- 파일럿 학원 1곳 확보 및 데이터 검증
- ClassIn 내부 이해충돌 확인 및 포지셔닝 정리

### 주요 리스크
| 리스크 | 완화책 |
|--------|--------|
| ClassIn API 변경 | Layer 1(API 래퍼) 분리 설계 |
| 학원 PC 상시 가동 불가 | Cloudflare Tunnel + 절전 해제 or 라즈베리파이 옵션 |
| 원장의 기술 문맹 | 대시보드 단순화 + 교육 패키지 포함 |
| 기존 학원 SW와의 경쟁 | ClassIn 결합이 유일한 USP 유지 |
